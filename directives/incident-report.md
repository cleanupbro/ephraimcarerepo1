# Directive: NDIS Incident Report

## Goal
Document and respond to an NDIS incident.

## When to Use
- Participant injury
- Safety concern
- Rights violation
- Significant event

## Steps

### 1. Immediate Response
- Ensure participant safety
- Provide necessary support
- Document initial details

### 2. Log in System
Via admin dashboard:
- /admin/incidents â†’ New Report
- Fill all required fields
- Severity: low, medium, high, critical

### 3. Notify Relevant Parties
High severity triggers automatic:
- Telegram alert to admin
- SMS to designated contacts

### 4. Follow-Up Actions
- Document actions taken
- Update incident status
- Mark follow-up required if needed

### 5. NDIS Compliance
Critical incidents must be reported to NDIS Quality and Safeguards Commission within 24 hours.

## Severity Levels

| Level | Definition | Response |
|-------|------------|----------|
| Low | Minor issue, no harm | Log only |
| Medium | Moderate concern | Log + notify admin |
| High | Significant event | Log + notify + immediate response |
| Critical | Serious harm | Log + notify + NDIS report + escalate |

## Documentation
All incidents logged in Supabase for NDIS audit compliance.
